📌 Project Overview

This project is part of the Open Source Software course. It helps us understand the philosophical aspects of open source. The main goal is to analyze Git and show Linux skills through shell scripting.

Git is a distributed version control system. It helps developers track changes work together and keep a history of their projects.

This project also includes five shell scripts. They automate Linux tasks like system inspection, package verification, disk analysis and log monitoring. These scripts show how open-source tools can be used in real-world environments.

🎯 Objectives

* To understand open-source software

* To explore Git as a world open-source tool

* To gain hands-on experience with Linux. Shell scripting

* To demonstrate automation using Bash scripts

* To connect knowledge with practical implementation

⚙️ Scripts Description

🔹 Script 1: System Identity Report

This script gives a summary of the system environment. It shows:

* Linux distribution and kernel version

* logged-in user

* System uptime

* Current date and time

👉 It helps in understanding the system configuration and environment.

🔹 Script 2: FOSS Package Inspector

This script checks if Git is installed on the system. It also:

* Displays the installed version

* Shows package-related details

* Provides a description

👉 Useful for verifying software installation and understanding package management.

🔹 Script 3: Disk and Permission Auditor

This script analyzes system directories and reports:

* Disk usage of each directory

* File permissions

*. Group details

👉 Helps in understanding storage usage and system security.

🔹 Script 4: Log File Analyzer

This script reads a log file.

* Counts occurrences of a keyword (e.g., ERROR, WARNING)

* Displays summary results

👉 for debugging and monitoring system activity.

🔹 Script 5: Open Source Manifesto Generator

This interactive script:

* Takes user input

* Generates an open-source philosophy

* Saves the output in a file

👉 Demonstrates creativity and the philosophical aspect of open source.

▶️ How to Run the Scripts

Step 1: Give execution permission

chmod +x script1.sh script2.sh script3.sh script4.sh script5.sh

Step 2: Run scripts individually

./script1.sh

./script2.sh

./script3.sh

./script4.sh <logfile> [keyword]

./script5.sh
